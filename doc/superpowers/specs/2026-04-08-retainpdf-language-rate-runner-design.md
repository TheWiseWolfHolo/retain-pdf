# RetainPDF 目标语言 / 并发限流 / 一键启动 设计说明

## 目标

在保持当前浏览器端风格统一、简洁、优雅的前提下，为 Docker 版 RetainPDF 增加四项能力：

1. 目标语言选择
2. 并发数控制
3. 请求速率限制
4. Windows 下一键启动 / 停止 / 重启

## 设计原则

- 不把主界面变成工程参数面板
- 常用项直接可见，高级项折叠收纳
- 真正把参数打通到后端与翻译链路，不做假开关
- 限流在模型请求底层统一生效，覆盖翻译、重试与修复类请求

## 前端设计

浏览器“接口设置”弹窗新增一个“翻译偏好”卡片：

- 直接可见：
  - 目标语言
- 默认折叠的高级控制：
  - 并发数
  - QPS 上限
  - RPM 上限

所有值写入浏览器本地存储，并同步到隐藏字段，提交任务时进入 `translation` payload。

## 后端与脚本设计

Rust API 的 `CreateJobInput.translation` 新增：

- `target_language`
- `rate_limit_qps`
- `rate_limit_rpm`

命令构建层把这几个字段透传到 Python CLI。

Python 侧：

- 在翻译入口统一配置请求限流
- 在翻译 prompt 构造时注入“目标语言覆盖”说明
- 公式窗口翻译与乱码重建同样使用目标语言

## 一键启动设计

仓库根目录新增：

- `启动 RetainPDF.cmd`
- `停止 RetainPDF.cmd`
- `重启 RetainPDF.cmd`

脚本固定走当前本地增强版镜像标签与已验证端口，并在启动后自动打开 `http://127.0.0.1:40001`。
